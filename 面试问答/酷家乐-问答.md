**面试官：** 再问一些 React 应用方面的问题吧。React 的 useCallback 呀、useMemo 啊，它们通常都会有什么作用呢？

**回答：** 主要是一个性能优化的函数。useCallback 是缓存函数，useMemo 是缓存值。useMemo 的话，比如这次会从一个后端返回很多数据，需要进行大量计算，可以把值缓存起来。useCallback 是因为经常会用到回调函数，React 的特性是单向数据流，每次重新渲染会把当前组件全部重新执行一遍。为了避免重复渲染，就会用到 useCallback 和 useMemo，它们可能作为下一个组件传入的 props，再配合 memo 函数进行性能优化，避免重复渲染。

**面试官：** 你们现在需求是大屏比较多，还是移动端的小屏比较多？

**回答：** 大屏是我之前临时支援的，一般都是做支付宝运动那块的 SDK 和传感器授权、弹窗。现在小程序和 H5 比较多。

**面试官：** 它这个拉取新资源的时候，有时候白屏的时间会比较久。在减少白屏时间方面，你有没有什么具体的经验？

**回答：** 白屏主要是两个点，一个是资源请求太久了，一个是没命中缓存。具体的话，可以把多个链式请求合并成一个，用 Promise.all 同步请求下来，快速把重要的主接口数据查询到。还有像 React 这样的库，用强缓存缓存下来。头图和固定位置的图片也可以用 CDN 加缓存的方式。还有预加载，在容器启动 WebView 的阶段提前发起请求，业务代码挂载的时候拿到之前的请求。小程序也会把重要数据全部缓存下来，第二次进来时先把缓存加载出来直接渲染，再进行异步更新。还有骨架屏的方式让用户感觉页面在加载。

**面试官：** 我看你简历中有用到什么 CSS Module 去做样式隔离。

**回答：** 哦，对。

**面试官：** 你可以说一下这个 CSS Module，它实现样式隔离的原理是怎样的吗？

**回答：** 原理是会用到打包工具的一个配置。它会根据 CSS 文件所在的位置或随机数，在你写的样式后面加一些哈希值的后缀，保证 class 属性的唯一性，达到样式隔离的效果。

**面试官：** 那它在编译和构建的过程中还需要做哪些事情？你对打包工具有没有一些实践？

**回答：** 在打包中就是打包构建中实现的，主要用 PostCSS。以 Webpack 举例，它会有 loader 的过程，把 CSS 代码转换为 JS 代码让 Webpack 识别成一个模块。在这个过程中，Webpack 可以把它解析成 AST 语法树。还有插件机制，整个是一个发布订阅模型，在整个周期内释放出钩子函数。我之前有写一个小的 demo，在生产环境把 console.log 代码全部剔除掉的小工具。
