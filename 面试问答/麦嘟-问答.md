## 面试复盘摘要

### 岗位匹配
麦嘟的岗位是 AI Agent 方向，面试官说前期会用 AI 开发现有的 H5 迭代，后期针对公司自己的业务垂类做 AI 功能调优和开发。和我转 AI Agent 的方向一致，但感觉这家对 Agent 的落地还比较早期，更多是"用 AI 写代码"而不是"开发 AI 产品"。公司产品线较杂（多个 App），技术栈偏传统。匹配度中等。

### 面试问题
- **自我介绍**：主动表达想转 AI Agent
- **前端基础**：深拷贝/浅拷贝、Vue diff 算法
- **React 概念**：声明式编程（答不上来）
- **AI 实践**：Claude Code + MCP + Skill、sketch MCP、组件代码生成
- **规范开发**：openSPE、SDD
- **职业规划**：想转 AI 全栈
- **反问**：岗位方向、团队规模

### 表现好坏
✅ **答得顺的**：深拷贝/浅拷贝基本概念清楚，Vue diff 算法简单说对了方向。AI 实践部分（Claude Code + MCP + Skill）讲得比较丰富，有实际经验支撑。职业规划表达清晰。
❌ **卡住的**："声明式编程"这个基础概念完全没答上来，这不该是前端开发者说不出的概念。面试时间很短（不到 15 分钟），整体感觉面试官没有深入追问的兴趣。反问环节聊得比较浅，没有展现对公司的深入了解。

### 面试官关注点
面试官似乎更关注 **前端基础能力**（深拷贝、Vue diff、声明式编程），对 AI 实践的追问不够深入。整体感觉面试流程比较形式化，可能岗位本身定位不太清晰（既想做 H5 又想追 AI）。面试时间短，没有给太多展示机会。

### 下一步改进
- 基础概念必须补："声明式编程 vs 命令式编程"——React 的核心就是声明式 UI（你描述 UI 应该是什么样子，React 负责更新 DOM），这是面试必考题
- Vue 相关的基础概念要整理一份速查表（diff 算法、响应式原理、生命周期等）
- 面试时间短的情况下，要主动引导话题到自己擅长的领域，不要等着被提问
- 求职定位要清晰，自我介绍时就把"我想做什么、我擅长什么"说清楚，避免浪费前面几分钟

**面试官：** 行，那就问你一下这个 Vue 的 diff 算法是什么原理？
