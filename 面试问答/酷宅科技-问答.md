**面试官：** 问你一些 React 相关的问题。你用过的一些 hooks 都有哪些，具体功能是什么？

**回答：** 最常用的是 useState，管理状态，用来更新页面。还有 useEffect，主要用于数据请求。它把之前的生命周期收敛到这个 hook 里面，还有一个依赖数组，根据数组里面的值决定是否执行回调函数。一般用于数据请求，还有动态监听一些值的变化来完成页面更新。还有 useMemo，缓存一个计算值。如果依赖的值需要进行大量计算，可以用 useMemo 缓存起来。如果某个值会传到下一个组件的 props 里面，引起下一个组件更新，可以用 useMemo 缓存，下一个组件用 memo 包起来。还有 useCallback，跟 useMemo 类似，是缓存一个函数。常用的就这些。

**面试官：** useEffect 里面的第二个参数，依赖数组，会有哪几种情况？传递之后具体是什么效果？

**回答：** 如果不传，每次更新都会重复执行，一般很少这样用。还有一种是空数组，组件首次渲染时执行，后面不再执行。还有传递值，可能是普通类型或引用类型。普通类型更新后如果值没变就不会重复执行，引用类型如果改变就会重复调用回调函数。

**面试官：** useCallback 跟 useMemo 有什么区别？

**回答：** useCallback 是缓存函数。比如有一个请求函数，可以用 useCallback 包裹起来，放到 useEffect 里面进行首次调用，后面还可以用返回的函数在其他地方手动调用。useMemo 是缓存计算的结果值，比如计算量大、后端返回数据需要处理，或者其他复杂状态变更，可以放在 useMemo。主要区别就是一个缓存值，一个缓存函数。

**面试官：** useCallback 跟普通的 callback 有什么区别？

**回答：** 普通 callback 在组件重新渲染时，整个组件都会重新执行，函数会重新创建，有新的引用地址。useCallback 则不会，函数对象没有被销毁。

**面试官：** 你能说一下你了解哪些吗？

**回答：** 主要了解 React 16.8 之后引入了 React Fiber 架构。它通过自己实现的调度规则，把组件分成一个个小的工作单元，可以并发执行。Fiber 其实是双链表结构，一个是工作中的树，一个是已经渲染的树。如果页面状态更新，会动态计算这棵树，通过 Fiber 一个个工作单元进行比较更新。工作单元用自己实现的调度器，在每一帧里面调用一定次数的更新比较。如果中间出现优先级更高的任务，比如用户输入操作，会停下来。因为是链表，用户结束高优先级任务后，会从这个链表恢复过来，继续往下比较。这是渲染过程。后面如果全部比较完，发现哪里有更新，统一一次 commit，把虚拟 DOM 变成真正的 DOM，一次性提交渲染到页面。提交过程是不可中断的，render 过程可以中断。大概就是这些。

**面试官：** 时间分片有了解吗？

**回答：** 时间分片其实就是我刚才说的调度器。它通过浏览器的一个 API 创建一堆微任务，把很多 Fiber 工作单元放到每一次帧中执行。如果有更高优先级任务插进来，会先把其他执行完，再回来执行刚才没完成的工作。时间分片实现了任务可中断、可恢复的能力。

**面试官：** 如果在浏览器上输一个网址，点回车之后，浏览器会做哪些事情？

**回答：** 输入网址后，会通过 DNS 解析到 IP。拿到 IP 后进行 TCP 连接。在这之前可能有浏览器缓存或 HTTP 缓存，判断强缓存和协商缓存策略。如果命中强缓存，直接拿到数据。如果是协商缓存，通过 ETag 等标识校验，返回 304 状态码。普通情况下会拿到 HTML 文件，比如 SPA 应用只返回一个 HTML 文件，里面包含 HTML 元素、CSS 样式文件、JS 文件。浏览器同步加载这些静态文件，碰到 JS 会先执行，如果有 defer、async 等标记按特定策略加载。加载完成后渲染 DOM 树，CSS 渲染 CSSOM 树，后面统一绘制渲染。渲染结束后，如果 JS 对 DOM 有更新，会进行重排重绘，更新页面。
