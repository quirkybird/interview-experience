# 超线性 - 面试问答

---

## 面试复盘摘要

### 岗位匹配
超线性是一家小公司，主要做 AI 搜人，场景包括招聘（对标 LinkedIn）和网红数据库搜索。目标是比传统搜索效果更好、更自动化，挖掘长尾匹配。岗位本身更偏前端，但公司小、分工不固定，会看个人能力和潜力往 AI 或全栈方向发展。和我想要的 AI Agent / AI 前端方向有一定匹配，但不是纯 Agent 研发，更偏业务前端 + 可能参与 AI 搜索。

### 面试问题
- **项目背景**：阿福智能体与运动健康模块的关系、与阿福 APP 的共通能力
- **RAG 架构**：为什么自建向量库、如何与通用能力融合、agent 如何选知识库
- **Agent 流程**：意图识别层 + agent 层双层召回、整体运行流程
- **卡片交互**：用户 / agent / 后端数据流转、卡片如何被召回、渲染后交互事件
- **LangChain**：核心解决的问题、是否多智能体
- **个人情况**：工作年限、求职预期、离职原因

### 表现好坏
✅ **答得顺的**：对项目整体架构（RAG、MCP、卡片）有一定理解，能讲清楚数据流转和双层召回的设计思路，也能解释意图识别召回为何比直接走 agent 更快。
❌ **卡住的**：面试过程中网络卡顿影响节奏；对 LangChain 核心解决什么问题表述不够清晰，回答偏"封装 SDK"而不是"链式调用、模型抽象、工具调用"；多智能体概念被自己说混了（多个模型 vs 多 agent）；对 agent 落地模块的回答比较零散；TCP/SSL 这类基础网络问题没问到，但如果问到估计也是薄弱点。

### 面试官关注点
面试官非常在意**系统架构的清晰度**和**数据流转的准确性**，追问得很细（message 怎么给模型、是否模拟 assistant 输出、卡片交互事件是否回传模型）。他想知道我对这套 AI 系统是真的理解还是只会讲概念。

### 下一步改进
- 准备一套标准化的项目架构描述：用户 query → 意图识别 → agent → 工具调用（RAG/MCP）→ 结构化输出 → 前端渲染
- 明确 LangChain / LangGraph 的核心价值：链式组合、模型抽象、工具调用、结构化输出
- 区分"多个模型"和"多智能体"，避免概念混淆
- 对模拟 assistant message、上下文管理这些工程细节要能自洽地讲清楚
- 面试前检查网络，避免卡顿影响沟通

**面试官：** 可以听到吗？不好意思，刚刚有点事在开发。那我就开始，麻烦简单介绍一下。

**回答：** 嗯，面试官你好，我叫杨青宇，毕业于渤海大学软件工程专业。之前是在数字马力担任前端开发，主要负责支付宝健康事业部相关的业务开发。之前的工作主要分为两部分：一部分是传统业务开发，包括医疗健康小程序、健康岛、支付宝运动等相关业务。我参与了整个健康能量体系建设，它是一个兑换媒介，包括权益兑换整个链路和权益中心，还有一些高频 DAU 的核心功能开发，同时也负责一些通用组件和 SDK 的抽象沉淀。第二部分是 AI Agent 的相关开发，也是我工作一年后投入比较多的方向。我深度参与了阿福人体运动健康模块，从前端到后端都有涉及。技术栈主要包括 React、Next.js、NestJS、LangChain、MongoDB、Milvus 这些。在这个项目里，我负责搭建 RAG 的运动健康知识系统，对一些问答、规则、权益信息进行向量化处理，并基于 Milvus 实现检索能力。同时还封装了多模型调度层，切换不同模型，还包括一些 MCP 工具，让智能体能够在对话内获取业务数据，实现 SSE 流式输出和多轮对话。除此之外，我也比较关注性能优化、打包优化，以及在工作中使用 Web Coding 的一些实践。后面希望能够继续在 AI、AI Agent 这方面深入发展。以上就是我的基本情况。

**面试官：** 好的，先了解一下你们的业务背景。你这个智能体跟阿福那个 APP，它是什么关系？

**回答：** 它是支付宝里面的阿福下面的一个模块。

**面试官：** 相当于是阿福 APP 那个智能体吗？

**回答：** 它其实是多端的，有一个阿福 APP，然后在支付宝里面还有一个阿福。它是用 Web 这些技术栈搭建的。

**面试官：** 也就是说，如果品牌是一样的话，你们和阿福 APP 之间有哪些能力是共通的？

**回答：** 共通的是一些基建能力，比如调用支付宝已有的 SDK，像 Galaxy JSAPI，还有一些医疗知识问答都是通用的，基建能力是通用的。

**面试官：** 既然你们跟他们是一个系统，为什么还需要自己自建向量库这些东西？

**回答：** 因为这是我们业务自己有一些特定的业务。比如建向量库，是因为我们这边有权益中心，去兑换一些东西，这些页面之前的业务都是这些。还有运动里面本身有一些规则、活动，还有一些常见的客服问答，所以就需要新建一个特定业务定制化的东西。

**面试官：** 那你这个定制化这一层，怎么跟通用的那一层去做融合？

**回答：** 这个相当于我们这边提供一个 RAG 的能力，最后把 RAG 封装成一个内置工具。要么就是意图识别到使用这个工具，要么就是 agent 自己判断是否需要调用这个工具。

**面试官：** 我的意思是，你们这个 agent 如何去判断它应该用哪一个知识库？因为按我的理解，它首先在阿福的智能体有它一套基建，然后你这边又定制了一套东西。

**回答：** 它大概就是很多层，有意图识别层、模型回答，最后输出是编排层。每个智能体有不同的 type ID，相当于每个智能体不一样，然后它就可以返回特定的一些装载特定工具，还有一些特定的业务逻辑。

**面试官：** 那你把这里结构化的描述一下，整体来说应该是怎么样一个运行流程。

**回答：** 整体就是我们开启子智能体的时候，包括医生这些，跟我们运动健康模块一样，是子智能体。子智能体的话，通用基建能力是共用的。但针对每个业务，你肯定可以搭建一套自己的 agent，内部封装了 LangChain、LangGraph 这些。然后我们在自己业务里面跑起来，定制一些自己的业务逻辑。

**面试官：** 你的这个业务逻辑就是一些知识性的问答吗？还有别的能力吗？它能够操作用户的数据吗？

**回答：** 可以。我刚才说了，我们还有内建一些 MCP，比如我们之前有健康能量，像森林能量这样的媒介，去兑换一些东西，可以在不同场景里面获取。之前是通过页面点击特定页面，然后走兑换链路。现在可以在这里面通过 MCP 封装的接口拿到这个信息，然后弹出一个卡片，用户可以用这个路径去兑换。比如可以拿到用户的余额、状态，动态判断可以召回哪些可以兑换的商品，然后在这个页面进行兑换。还有比如运动计划之类的，反正就是通过一些特定组件渲染出来，然后进行商品兑换。

**面试官：** 这些卡片的开发是你自己在做，还是你们团队其他人在做？

**回答：** 我是开发了一些具体的卡片。

**面试官：** 你们这个分工是别人搭一个框架吗？

**回答：** 对，相当于我们开发完卡片之后，需要在一个系统里面把这个卡片注册好。卡片其实就是一个组件，有很多个组件，注册好之后，业务方或者运营会把提示词对应的卡片、什么时候命中这个卡片定义好。然后后端会返回特定的一些数据，再渲染这个卡片。我可能就是开发具体的卡片，涉及到一些前端展示的业务。

**面试官：** 那就详细讲一下这个卡片的交互逻辑。这里面涉及三方：用户、agent、后端。数据是怎么流转的？

**回答：** 我们开发一个卡片之后，卡片所有的数据都是通过 props 传进来的。数据是后端发给我们的。具体为什么要出这张卡片，是通过提示词和意图识别，运营会配置。意图识别会识别到这张卡片，然后对应走后端的路由逻辑，请求对应的 RPC 接口或者服务。最后通过一个流程编排，统一走结构化输出。然后数据传给前端，通过 props 传给匹配到的组件，最后渲染成一条消息给用户。

**面试官：** AI 是怎么知道需要用到这张卡片的？

**回答：** 它其实有两层。一层是意图识别，比如产品给了一些关键词，我们会在系统里注册这些关键词。用户发 query 的时候会 embedding 匹配这些意图，如果直接匹配到了，就直接把卡片数据从后端带回来。如果没有意图匹配漏掉，我们是一个 ReAct agent，系统会提前把卡片信息发给 agent，agent 自主判断是否需要召回这个卡片，然后走对应的后端链路。

**面试官：** 为什么要做成两层呢？为什么要先做一层意图识别，再做一层 agent？

**回答：** 这个也是时间和成本考虑。意图识别可能是基于算法和语义识别，比较薄的一层。如果能从这里直接识别到用户意图，就不会走 agent 自主判断调用大模型这一步，体感会比较快。如果用 agent，相当于调用模型，模型再进行判断，就比较慢了。而且可能也达不到对应的效果。卡片召回可能是比较强意图的才会召回。

**面试官：** 如果从用户来说，意图识别召回和 agent 自主召回在同一个对话流里，他是没有感知的。但对于 agent 来说，它是否知道前面的聊天记录？比如第一个问题是意图识别直接召回的，第二个问题是 agent 自己回答的，agent 是否知道前面的聊天记录？

**回答：** 这个属于上下文的东西。我们肯定会记录，比如发下一条信息会有一个会话窗口，会带这十条信息，加上之前的一些摘要。虽然内部没有走到 agent 这一步，但我们会根据消息、长期记忆、之前对话摘要发给 agent，agent 是能感知到上下文的。

**面试官：** 具体一点，这种意图识别召回的结果是怎么交给 AI 的？是作为聊天记录吗？

**回答：** 对，就是作为聊天记录。

**面试官：** 那这句话在 message 里面体现为是模型自己说的话吗？

**回答：** 它肯定是一些结构化的数据，是一些 metadata。

**面试官：** 我关心的是它到底是谁说的话。正常来说，大模型的 message 要么是 user 发的，要么是 system 发的，要么是 AI 自己说的话。你刚才说你模拟了一次 AI 的输出，但实际上大模型自己没输出过这句话。你给 message 的时候，是把这段话当成它自己说的吗？

**回答：** 对，就是模型说的话。相当于让它知道之前拿到过这个信息。一般就是 user、assistant 对话，还有一些系统消息。

**面试官：** 也就是说，对于意图识别这种模式，它也是模拟了一次从聊天记录里面模型自己说的话。

**回答：** 对，assistant 自己说的话。

**面试官：** 那卡片渲染出来之后，用户交互包括更新会怎么样？

**回答：** 卡片里面有不同类型，比如可能就是正常业务里面的 banner，或者里面有一个大 banner 加很多服务。基本上就是之前的前端逻辑，加上一些跳链，可能会跳到其他页面。也有点某个交互会再给模型发一条消息，相当于一个快捷输入。还有拉起弹窗这些。

**面试官：** 这些交互事件，除了直接跟模型对话的那种，其他的是否还会给模型发过去？

**回答：** 比如跳链就是跳走了，肯定就没有交互。如果有一些快捷的，点卡片里面某些东西会主动再给模型发一条消息，这个消息就是普通消息。

**面试官：** 那这个产品流程就是，用户实际要完成的事情还是要自己点卡片才能完成，agent 不能完全帮他完成这件事，是吧？

**回答：** 对，我们这个是比较轻的，没有帮他创建订单、自己购买这些流程，我做的比较薄。

**面试官：** 刚刚有点像偏智能一点的导航，差不多是这个意思吗？

**回答：** 对，有点像智能客服，能问一些规则、权益信息，比如现在可以兑换哪些东西。

**面试官：** 那这个 AI 是你自己深度参与开发的吗？就是 LangChain 和 LangGraph 这套东西。

**回答：** 对，我主要是用 LangChain 比较多。

**面试官：** 那你稍微讲一下 LangChain 核心解决了什么问题吧。

**回答：** LangChain 主要解决的问题是它是一个链，可以把 prompt、结构化输出、pen 这些串起来。从 prompt 到中间插入很多处理，最后输出。它还相当于一个 SDK，会抹平不同模型 SDK 的差异。我们其实没有那么复杂，就是调用里面的 ReAct agent 接口，它会处理中间所有东西，返回内容包含不同 type，比如 AI 消息或者工具调用结果。我们把这些信息用 SSE 返回给前端，前端通过不同类型和数据渲染不同内容。后面也是把 RAG 做成工具，让它在里面调用。

**面试官：** 你这个里面涉及多智能体吗？

**回答：** 单智能体，没有拆分子任务然后多智能体并发。

**面试官：** 你前面自我介绍里提到参与多个智能体落地，这个多个智能体是指什么？

**回答：** 多个智能体是多个模型提供方案，不是说多 agent。比如千问、疗养模型、DeepSeek 这些。

**面试官：** 那你这个项目落地时大概会涉及哪些模块？

**回答：** 整体上来说需要一个 prompt，不同问题用不同 prompt。还有一些工具、MCP、结构化输出，不同 type 需要不同结构化输出，比如工具调用会用到 JSON。还有意图识别、召回逻辑这些。

**面试官：** 你现在是毕业一年多吗？

**回答：** 2025 年毕业的，涉及到 AI 的部分大概是半年时间，可能也不是很深。

**面试官：** 现在找工作的预期是什么？

**回答：** 想找一个 AI Agent 相关的，全栈 AI Agent 开发，主要还是想找 Agent。

**面试官：** 现在已经离职了？

**回答：** 已经离职了，上个月离职。

**面试官：** 能了解一下离职的原因吗？

**回答：** 我之前主要是做支付宝运动相关的，现在业务虽然有阿福智能体，但只是里面的一个单独的东西。业务那边有收窄，支付宝运动、健康这些全部偏维护状态。考虑到职业发展和现在的大趋势，想出来找新机会。

**面试官：** 我这边没有其他想了解的，你有什么想聊的吗？

**回答：** 我们这边主要是做什么产品？

**面试官：** 主要能看到的产品是用 AI 组来做搜人这件事。市面上非 AI 模式的搜索，招聘场景下海外用 LinkedIn，网红相关有专门的数据库和 SaaS 去搜。但一个数据用起来效果不好，另一个不够自动化。我们现在主要目标是用 AI 把搜索效果做得更好、更自动，挖掘更长尾的匹配。AI 会更深入帮你挖掘需求，匹配对应的人。整体简单来说就是做 AI 搜索这件事。

**回答：** 那这个岗位具体进去会做哪些事情？

**面试官：** 看个人能力和潜力。因为一年的话，对我们来说相当于跟应届生差不多，所以会更看个人。公司小，不会像大公司有那么具体固定的岗位划分，你在哪个地方发挥比较好，就会让你更多去做那块。所以需要一个能接住业务需求的前端。一开始还是前端，但能做一些后端开发也可以。

**回答：** 了解了，没有其他问题了。

**面试官：** 好，那就先聊这么多。

**回答：** 好的，拜拜。
