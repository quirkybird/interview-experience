这场是密集的前端基础八股 + 图像处理场，几乎没问我的 RAG/Agent 深度：

**面试官：** ES6 里面你觉得哪些特性是做 Vue 或 React 时经常用到的？

**回答：** 比较多的比如声明变量 let 和 const，让它有块级作用域、避免全局污染；箭头函数，写起来比较简单，也没有 this 的问题；模板字符串，拼接字符时不用字符串相加，直接用变量拼接；展开运算符，三个点的形式可以很快组合一个对象或数组；对象解构，能直接把对象里面的内容拿出来，还可以给默认值，数组解构也一样；还有异步编程的 Promise，它有三个状态 pending、resolve、reject，状态不可逆，经常用在网络请求、并发请求 Promise.all 这些。

**面试官：** 那 TS 里面有哪些你觉得跟 JS 差别比较大的特性？

**回答：** 最基本的是能进行类型校验，配合 ESLint 能检验类型错误，在编辑器里就会爆红，在开发阶段就能知道参数错误。围绕类型它还有泛型的工具类，能动态传入类型，比如一个返回值是 Promise、值不确定，可以用泛型传入一个类型动态改变返回类型。还有一些好用的内置工具类型，比如 Omit、Partial 这些。主要是用它的类型比较多。

**面试官：** 那前端跟后端的网络请求一般是怎么实现的？

**回答：** 前端现在主要用 XHR、fetch 这些接口，把服务器的域名地址填进去，get 一般把参数拼装在链接上，post 一般有一个 body 请求体。发给后端之后，后端监听端口，一般用 Nginx 进行转发——服务器上可能有很多不同的服务，Nginx 把请求转发到对应的服务上。比如 NestJS 这块，服务进来之后会先经过前置的拦截器、守卫，做权限校验，把前端的一些请求头取出来解析到用户信息，再去对应的后端查询数据库，最后返回给前端。前端拿到状态码进行解析，如果是渲染内容就解析渲染。

**面试官：** 那前端怎么去异步上传一个文件到服务器？

**回答：** 前端调用 FormData 这种 content-type 传给后端。如果要做异步上传，就是把文件内容用 buffer 分成很多块，相当于分片上传，把这些分片同时传到后端。因为每个分片上面有 ID，包含它的顺序，后端再对最新的片段进行记录，并对这些片段重新组合成原文件。后端会实时告诉前端状态，比如你有 500 个分片，已经上传了 250 个就知道是 50%，可以做进度通知。上传结束之后后端给前端一个标识，表示文件上传成功，可以返回文件的地址之类的。

**面试官：** 除了刚才讲的这两种数据交互，还有其他方式跟后端交互吗？

**回答：** 有，比如做一个实时编辑器可能会用到 WebSocket，这是全双工的通信，后端起一个 WS 服务，它基于事件，前后端都可以互传内容，一般会用心跳检测确保双方一直建立连接，断开可能会有重试。还有现在 AI 用得比较多的 SSE，但一般不会用原生的 SSE，因为原生 SSE 只支持 GET 请求而且只是浏览器支持，我们一般用 POST 加 content-type 为 text/event-stream，用 ReadableStream 的方式进行请求，这个还是 HTTP 这块的。WebSocket 是基于 WS 协议的，会有一个协议转换。我了解的请求一般就是这些。

**面试官：** 那你觉得近年来前端有新增了哪些比较新的技术？

**回答：** 之前一直有虚拟 DOM 的概念，最近比如 Svelte，还有 Vue 最新的一些实验版本，也支持无虚拟 DOM，这可能是比较新的。还有服务端渲染的技术，React 的 Next.js、Vue 的 Nuxt.js；还有 SSG 方面的技术，比如 Astro，它适合做文档、博客、官网，在构建过程中静态已经帮你解析成了 HTML 文件，服务端请求时直接返回这个文件，非常快。前端我大概就能想到这些。
